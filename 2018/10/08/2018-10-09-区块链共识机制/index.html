摘要：了解区块链的共识机制和比特币原理。

区块链共识机制

比特币历史背景

起源于密码朋克，诞生于2009年，由中本聪提出，是目前为止（2018）市场总值第一的数字货币。有着匿名，分布式，不可篡改，一致性，全球化等特性。

比特币的基本服务：

  • 身份确认

    用户的地址携带用户的账号信息，是匿名的。

  • 交易服务

    交易发生在用户之间，不经过第三方 机构（如银行）。

  • 记录服务

    通过区块链技术在数以万计的用户间 存储交易信息，通过信息冗余及高额造假成本确 保安全 。

  • 信任规则

    个人激励机制，全网分布式账本。

比特币共识机制

比特币的共识机制就是从比特币提供的基本服务出发，对每种服务进行具体的讨论。

  • 比特币身份确认

    • 公钥和私钥

      身份确认用于表示自己账号，比特币使用公钥和私钥作为身份的确认的基本元素。每个账户可以被唯一一个公钥标识，简单地说，公钥相当于比特币的保险柜，而私钥相当于钥匙，私钥是账户拥有者持有。

    • 私钥，公钥和地址之间的关系

      先通过随机数生成用户私钥，私钥经过不可逆的椭圆曲线加密算法生成公钥，公钥经过两次哈希生成公钥哈希，加上版本号进行一系列处理以后形成钱包的地址。 可以说，私钥决定了账户的所有权。私钥有密码学的基础，从公钥得到私钥几乎不可能。

  • UTXO交易模型

    • 如何确认一笔交易的有效性

      • 所有权确认（私密->签名）

        仅仅有转账数额和转入钱包的地址无法确认交易的有效性，需要转出者提供相应的证明，唯一能证明的是私钥，所以带有私钥特性的签名对交易进行确认。

        一笔交易包含以下东西

        所有权的确认通过特定算法进行

      • 有可以进行交易的资金

        比特 币使用了Unspent Transaction Output (UTXO) 来确保同一笔资金 只出现在一笔交易中。UTXO的特点就是每一收到的资金都存 在一个储蓄罐里，每次交易都 需要打碎一个或多个储蓄罐。也就是说，有输出的罐已经没有余额，只有那些没有输出的罐有余额，当进行一笔交易时，取一个或者多个罐，将其额度取出，进行完交易后将剩下的比特币放在一个新的罐中，并且这个罐没有输出。

        1. UTXO基本模式：

          版本号+Input 未花费交易UTXO+Output输出地址及数量+锁定时间

        2. 安全性

          由于有所有权的确认，通过追溯，可以找到每个比特币的起源。

        3. hs

      • 可用的资金与其他交易的资金不冲突

  • 交易信息记录

    通过分布式数据库，每个节点都存储完整的账本。

    • 信任规则

      双花攻击就是一个用户用同一笔比特币完成两笔交易，这个攻击出现在交易进行确认之前。通过投票对交易进行确认并接入交易。

      多重身份攻击：当引入投票问题解决双花攻击，随之产生另外一个问题，那就是用户创建多重身份对交易进行投票。解决的方案是引入资源进行投票，投票的权利来源于用户具有的资源。

  • 工作量证明（POW)

    通过消耗资源解决一个问题 获得投票资格（即记账权）。

    记账权必须通过花费计算资 源来获得，比如说通过蛮力 解决一个问题；通过记账奖 励鼓励投入资源，即使一个用户有多个身份，但他所具有的计算资源决定了他的投票的公平性。

    挖矿就是投入计算资源来争夺记账权，记账权获得有一定的奖励，包括比特币奖励和交易的手续费，这个过程是通过付出大量的计算量来解决一个难题，谁先解决谁获得记账权，也就获得相应的奖励。整个争夺记账的过程就是挖矿的过程，也就是比特币发行的过程。那么问题来了，难题该如何设置呢？难题应该具有以下特点：不容易完成，容易验证，工作过程公平，具有随机性。

    为了更好地解释挖矿的过程，我们有必要理解一下区块链的特点：

    • 结构

    • 哈希函数

      哈希函数将任意长度的串转化为固定长度的串，并且串之间有一个字符不一样，那么得到的哈希值也完全不一样

      具体的特点有：

      • 正向快速：给定明文和 hash 算法，在有限时间和有限资源内能计算出 hash 值
      • 输入敏感：原始输入信息修改一点信息，产生的 hash 值看起来应该都有很大不同
      • 逆向困难：给定特定hash 值，在有限时间内很难（基本不可能）逆推出明文
      • 冲突避免：很难找到两段内容不同的明文，使得它们的 hash 值一致（发生冲突

    • Merkle Tree

      通过哈希树的引入，使得区块链的数据传递没有那么庞大，交易得哈希不断往上哈希直到存在一个哈希值，这个哈希值就是区块链中存储的哈希值。

    • 挖矿过程

      参与者综合上一个区块的 Hash 值，上一个区块生成之后的新 的验证过的交易内容的Markle Root值，再加上猜测的一个随机数Nonce， 一起打包到一个候选新区块，让新区块的 Hash值小于比特币网络中给定的一 个数。这是一道面向全体矿工的“计算题”，这个数越小，计算出来就越难。 （基于Hash函数特性）。

      SHA(Markle Root+上一个区块Hash值+Nonce)<某个数：挖矿成功！  “0000000000000f15673f1354”“0”的数目体现了挖矿难度。

      简单地说，挖矿就是在猜测Nonce的值，将任何一串数据输入到SHA256将得到 一个256bit的hash值，这个值必须小于或等于给出的特定值。每个区块的奖励最初是 50 个比特币，每隔 21 万个区块自动减半，即 4 年时间，最终在2140年比特币总量稳定在 2100 万个。因此，比特币是一种 通缩的货币。由于 Hash 难题在目前计算模型下需要大量的计算，这就保证在一段时间内，系统中只能 出现少数合法提案。反过来，能够提出合法提案，也证明提案者确实已经付出了一定的工作 量。这也保障了，如果有人尝试恶意破坏，需要付出大量的经济成

      比特币的一种发现方式就是通过挖矿。

    • 最长链机制

      如果有两个节点同时完成POW，这个时候会出现分链，解决方法是用户自定选择最长的链排队。但这样会存在51%攻击。

      设想这样一个现实场景：  Alice 和 Bob之间使用比特币完成了一杯咖啡的交易，Bob在收到Alice的转账通知 (交易提交)，就给 Alice提供了咖啡。  Alice不想支付这笔钱，在开始之前他把区块里的这笔交易改成Alice转给自己的一笔交易了(更改很容 易，只要把接收地址和签名改掉即可)。  Alice开始尝试用这个伪区块进行计算(计算正确后这个快会被加入主块中)，因为拥有51%的算力， Alice比别的节点更容易优先计算成功，导致一个伪造的区块加入了主链。

POW机制应用与局限

  • 局限

    • 矿池集中

    • 自私挖矿

      也就是当自己挖出的链长度有绝对优势，先不公布。

    • 计算资源浪费

      每年耗大量计算一个无用问题的计算量。
